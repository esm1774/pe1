## 1. نظرة عامة

### الهدف
نظام "بيئة مدرسة ذكية" متكامل لإدارة كل جوانب التربية البدنية، بدءاً من الحضور والقياسات الجسمية، وصولاً إلى تنظيم بطولات معقدة (خروج مغلوب، دوري، مختلط[طريقة كاس العالم]) ومنح أوسمة رقمية، مع توفير واجهات خاصة للتفاعل بين المعلم والطالب وولي الأمر.

### الفئة المستهدفة
- **مدير النظام (Admin):** تحكم كامل بالمستخدمين، الصلاحيات، والجداول.
- **معلم التربية البدنية (Teacher):** إدارة الفصول، تسجيل النتائج، وتنظيم البطولات.
- **المشرف التربوي (Supervisor):** متابعة الأداء والتقارير بمستوى أعلى.
- **قائد المدرسة (Viewer):** عرض التقارير والإحصائيات فقط.
- **ولي الأمر (Parent):** متابعة مستوى أبنائه، تلقي التنبيهات، وعرض الشهادات.
- **الطالب (Student):** عرض ملفه الشخصي، إنجازاته، وموقعه في البطولات.

## 2. هيكل الملفات
تم الانتقال من البنية الأحادية إلى بنية معيارية (Modular) لتسهيل الصيانة والتوسع.

```
📁 pe-smart-school/
├── 📄 index.php               ← الصفحة التعريفية للموقع (Dynamic Landing Page)
├── 📄 app.html                 ← الواجهة الأمامية الرئيسية (SPA)
├── 📄 api.php                 ← الراوتر الرئيسي للـ API
├── 📁 api/                    ← وحدات الـ API المعيارية
│   ├── 📄 auth.php, dashboard.php, students.php, attendance.php, etc.
│   ├── 📄 notifications.php, badges.php, parents_admin.php
├── 📁 modules/                ← المحركات الضخمة للمدربين
│   ├── 📁 tournaments/        ← محرك البطولات (Brackets, Matches, Standings)
│   └── 📁 sports_teams/       ← نظام الفرق والقرعة
├── 📁 js/                     ← الملفات المنطقية للواجهة (v2.6)
│   ├── 📄 tournaments.js, sports_teams.js, notifications.js, badges_admin.js
│   ├── 📄 parent_dashboard.js, student_dashboard.js, etc.
├── 📁 includes/               ← الكود الخلفي المشترك
├── 📄 config.php              ← الإعدادات المركزية والدوال المساعدة
└── 📄 PROJECT_DOCUMENTATION.md ← هذا الملف
```

## 3. التقنيات المستخدمة

### Frontend (v2.6)
- **HTML5 & Vanilla JS:** الاعتماد الكامل على البرمجة الصرفة لضمان السرعة.
- **Tailwind CSS (v4):** التنسيق الحديث والديناميكي.
- **Fetch API:** للتواصل المتزامن مع الـ API.
- **Canvas/Drag-and-Drop:** لتصميم الشهادات التفاعلية.

### Backend
- **PHP 8.x:** مع استخدام الـ Singleton Pattern للاتصال بقاعدة البيانات.
- **Modular Routing:** استدعاء الوحدات ديناميكياً حسب الحاجة.
- **PDO:** للتعامل الآمن مع MySQL.

## 4. قاعدة البيانات

### الجداول الجديدة والمحدثة (v3.0):

#### `notifications` - التنبيهات
تُستخدم لإرسال إشعارات فورية لأولياء الأمور عن (الحضور، نتائج اللياقة، الأوسمة).

#### `parents` & `parent_students`
إدارة حسابات أولياء الأمور وربطهم بأبنائهم (عبر رقم الهاتف أو البحث المباشر).

#### `teacher_classes`
نظام "تعدد المعلمين": يسمح بتعيين معلمين محددين لفصول محددة (دائم أو مؤقت).

#### `badges` & `student_badges`
نظام التحفيز: أوسمة رقمية (مثال: بطل الركض، الملتزم) تُمنح يدوياً أو تلقائياً.

#### جداول محرك البطولات:
- `tournaments`: (الاسم، النوع، نوع الرياضة، الحالة).
- `tournament_teams`: (ربط الفصل أو مجموعة طلاب بالفريق).
- `matches`: (إدارة الجولات، الأقواس Bracket، النتائج، الوقت والملعب).
- `match_events`: (تسجيل الأهداف، الكروت، الإصابات).
- `standings`: (جدول ترتيب الدوري - Wins, Draws, Losses, Goals).

## 5. نظام المصادقة والأدوار

| الدور | الصلاحية |
|-------|----------|
| `admin` | تحكم مطلق + إدارة المعلمين + ضبط النظام |
| `teacher` | إدارة الطلاب بالفصول المسندة له + تنظيم البطولات |
| `supervisor` | صلاحيات المشاهدة الشاملة لجميع الفصول والمعلمين |
| `parent` | دخول خاص لعرض بيانات الأبناء فقط وتلقي الإشعارات |
| `student` | دخول برقم الطالب لعرض ملفه الشخصي وإنجازاته |

## 6. واجهة API المعيارية

يعمل `api.php` كموزع (Dispatcher) يقوم باستدعاء الوحدة المناسبة من مجلد `/api`.

**أمثلة لـ Endpoints الحيوية:**
- `action=notifications`: جلب/تحديث التنبيهات.
- `action=award_badge`: منح وسام لطالب.
- `action=tournament_save`: إنشاء أو تحديث بطولة.
- `action=parent_link_phone`: ربط تلقائي للأبناء بولي الأمر عبر الهاتف.

## 8. محرك البطولات (Tournament Engine)

هذا هو القلب الرياضي للنظام، ويدعم:
- **أنواع البطولات:** خروج مغلوب (فردي/مزدوج)، دوري مجموعات.
- **نظام الأقواس (Brackets):** توليد تلقائي للأقواس حتى 64 فريق.
- **مباراة المركز الثالث:** دعم اختيارى وتلقائي لمواجهات الخاسرين في نصف النهائي.
- **الجدولة الذكية:** تنسيق المواعيد لمنع تعارض الحصص وضمان راحة الفرق.
- **التفاعل:** زر "Like/Cheer" للفرق واللاعبين لزيادة الحماس.

## 9. نظام الفرق والقرعة

## 14. ملاحظات للمطور

1. **إضافة ميزة جديدة:** يفضل إنشاء ملف JS جديد في `js/` وربطه في `app.html` بدلاً من تضخيم الملفات القديمة.
2. **الـ API:** إذا كان الإجراء يتعلق بالبطولات، مكانه في `modules/tournaments/api.php` وليس المجلد الرئيسي.
3. **التجاوب:** استخدم فئات Tailwind (sm, md, lg) بدقة لضمان عمل الواجهات بسلاسة كـ Mobile App.
4. **الأمان:** تأكد دائماً من التحقق من `canAccessClass(id)` عند جلب بيانات فصل معين.
